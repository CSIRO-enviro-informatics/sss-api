Linked Data API

About

This Application Programming Interface (API) is designed to provide access to GA's Samples, Sites and Surveys Database. GA has over 3 million samples that are each identified by an International Geo Sample Number for Samples, a digital catalogue record of sites used by GA and its Surveys stored in an ARGUS database.

Usage

This API is a Linked Data API in that it provides information about Samples and the list of Samples (the Sample Register) according to Linked Data principles. These include the principles that data items should be identified with persistent URIs and that information should be provided in both human-readable and machine-readable formats. To these ends, this Linked Data API resolves URIs for Samples of the following form:

Sample

http://pid.geoscience.gov.au/sample/{IGSN}

GA's base persistent URI is used (http://pid.geoscience.gov.au), the sample class of item is indicated by the individual sample's IGSN.

Another form of access is to use the OAI-PMH - Open Archives Initiative - Protocol for Metadata Harvesting: an HTTP API for harvesting metadata for objects in XML. This form of access is a requirement of the IGSN community

Site

This system provides landing pages for digital catalogue record sites utilised by GA. It contains views according to models such as the National Environmental Monitoring Sites Register and the Geoscience Australia's Public Data Model ontology.

This Linked Data API resolves for Sites used by the GA with the following form:

http://pid.geoscience.gov.au/site/{SITE_NO}.

Survey

This system provides the landing pages for metadata about Surveys stored in GA's corporate database known as ARGUS. It contains views according to models such as GA'S ARGUS, GA'S Public Data Model, PROV-O and the Semantic Sensor Network Ontology.

This Linked Data API resolves for Surveys from the ARGUS database with the following form:

http://pid.geoscience.gov.au/survey/ga/{SURVEY_NO}.

OGC WMS Web Service

While this web page describes this system for a human to read, it's not so good for machines, so we also provide a machine-readable version. We have implemented, as a test, an OGC WMS web service's GetCapabilities function which:

Retrieves metadata about the service, including supported operations and parameters
